Data model (conceptual)

POC Master is the driving list: one row per proof account. Required logical columns: SID (Salesforce Account Id, 15 or 18 chars) and POC_Record_Name (human label for slicers and drillthrough titles). In Power Query, rename your workbook columns to match or build a mapping step.

Treat SID as the only join key to Salesforce for Task 2. All child queries should filter to the POC Id set (batched) and attach POC_Record_Name after the fact with a merge from POC Master.

SOQL patterns (Account-scoped)

Replace {IDLIST} with a comma-separated quoted Id list from each batch. Confirm the anchor field per object using child-objects-catalog.csv from the hub.

Standard child (AccountId)

SELECT Id, AccountId, Name
FROM Contact
WHERE AccountId IN ({IDLIST})

Activity (polymorphic WhatId)

SELECT Id, Subject, ActivityDate, WhatId
FROM Task
WHERE WhatId IN ({IDLIST})

Notes (polymorphic ParentId)

SELECT Id, Title, ParentId, CreatedDate
FROM Note
WHERE ParentId IN ({IDLIST})

FeedItem (Account posts)

SELECT Id, ParentId, Type, CreatedDate
FROM FeedItem
WHERE ParentId IN ({IDLIST})

Custom objects: use the confirmed lookup API name, for example WHERE Loan_Account__c IN ({IDLIST}) once verified in Object Manager.

M query library cross-reference

Update any sample that still references AccountId alone in the POC sheet to your real headers: keep a single RenameColumns step at the start of POC load so the rest of the model stays stable.
