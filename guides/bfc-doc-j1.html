Row-level security checklist

  • Author a SecurityMapping table: viewer email → allowed LOB (or “Admin”).
  • Roles in Power BI: one role per LOB plus an admin role that uses TRUE() filter.
  • DAX pattern: compare USERPRINCIPALNAME() to mapping, filter fact tables by LOB column present on User.
  • Test each role with “View as” before mid-review; capture screenshots for evidence.
