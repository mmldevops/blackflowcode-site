Purpose and scope

This guide defines how Task 2 (and shared Task 1 pages) should look and behave in Power BI so the suite reads as one TD-aligned product: restrained color, clear hierarchy, and consistent semantics for match rates, orphans, and API health.

Apply the same theme JSON to every semantic model in the workspace. Keep executive pages free of monospace SOQL; relegate technical strings to tooltips or a dedicated “Technical” page.

Typography

  • Font: Segoe UI at dataset default; do not mix decorative display fonts.
  • Report title: 18–20 pt semibold, brand dark.
  • Page section headers: 12–13 pt bold.
  • Body and axis labels: 10–11 pt regular, muted gray for secondary labels.
  • KPI values: 24–28 pt bold, dark text; keep sub-labels one size smaller.

Page layout grid

  • Canvas: 1280–1440 px logical width; 16 px gutters between visuals.
  • Top band: Optional 48 px title row (dark fill or white with bottom rule in brand red).
  • Slicers: Single horizontal bar under title; sync across pages for POC set, date range, and object.
  • KPI row: Six tiles maximum on executive summary; Task 2 prioritizes: POC account count, objects queried, overall match rate, orphan count, last refresh status, API error count.
  • Detail tables: Full width below fold; zebra striping off (use subtle row lines only).

Charts and KPI rules

  • Bar and column: Brand blue for primary series; brand red only for emphasis (for example, unmatched segment).
  • Match rate: Use horizontal bar or gauge with explicit 0–100% axis; never crop the axis to exaggerate.
  • Matrix (object coverage): Conditional formatting: green >80%, amber 30–80%, red <30% match rate; document thresholds in tooltip.
  • Line (delta): Neutral blue line, markers only if fewer than 30 points.

Accessibility

  • Maintain 4.5:1 contrast for text on white; avoid light gray smaller than 10 pt on white.
  • Do not rely on color alone: add icons or text cues (“High risk”, “OK”) next to conditional colors.
  • Tab order: slicers left-to-right, then KPIs, then primary grid.

Governance and export

  • Watermark: “Confidential — POC” on summary pages for mid-review.
  • PDF: Use a bookmark layout with 12 pt minimum on print-optimized duplicates of key pages.
  • Drillthrough: Account detail page receives SID from matrix; show POC Record Name from POC Master dimension only (no PII expansion beyond scope).
