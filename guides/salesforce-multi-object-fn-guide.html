Multi-object Salesforce pull — M function pattern (Power Query)

Hub

M query library (500) · Task 2 job aide

Goal: one orchestrator that loops many Salesforce objects (Contact, Opportunity, custom __c tables, etc.), runs SOQL per object, normalizes to a common shape, and unions or keeps separate tables — without duplicating OAuth and URL glue in every query.

1. Parameters you reuse everywhere

  • SF_Instance — text, e.g. https://myorg.my.salesforce.com
  • SF_ApiVersion — text, e.g. v59.0
  • SF_Token — from a single cached query SF_Auth (never paste secrets into M text; use gateway credentials)
  • POC_AccountIds — table of distinct Ids used to build batched IN (...) clauses

2. Object registry (table of what to pull)

Author a query SF_ObjectRegistry as a static or Excel-driven table:

ObjectApiNameSelectFieldsWhereTemplateAnchorField
ContactId, AccountId, LastNameAccountId IN ({IDS})AccountId
OpportunityId, AccountId, Name, StageNameAccountId IN ({IDS})AccountId
MyObject__cId, Loan_Account__c, NameLoan_Account__c IN ({IDS})Loan_Account__c

Replace {IDS} at runtime with a quoted Id list from each chunk (same pattern as Task 2 queries 241–280).

3. Core FN — one function per SOQL call

Expose a reusable function query SF_RunSoql that accepts full SOQL text and returns a table (not just raw records list), so the orchestrator stays simple.

let
  SF_RunSoql = (soql as text) as table =>
    let
      Auth = SF_Auth,
      Url  = Auth[Instance] & "/services/data/" & SF_ApiVersion & "/query/?q=" & Uri.EscapeDataString(soql),
      Json = Json.Document(
        Web.Contents(Url, [Headers=[Authorization="Bearer " & Auth[Token]]])
      ),
      Recs = Json[records],
      T    = Table.FromRecords(Recs)
    in T
in SF_RunSoql

4. Orchestrator — loop registry rows

For each object row, build SOQL, call SF_RunSoql, add constant columns SourceObject and optional BatchId, then Table.Combine. Chunk Ids with List.Split so each SOQL stays under length limits.

// Pseudocode shape — plug in your chunking list from POC_AccountIds
let
  Chunks = /* list of text: "'001...','002...'" */,
  Rows   = Table.ToRecords(SF_ObjectRegistry),
  Tables = List.Transform(Rows, each
    let
      soql = "SELECT " & [SelectFields] & " FROM " & [ObjectApiName] & " WHERE " &
             Text.Replace([WhereTemplate], "{IDS}", Chunks{0}),
      T    = SF_RunSoql(soql),
      T2   = Table.AddColumn(T, "SourceObject", each [ObjectApiName], type text)
    in T2),
  Out    = Table.Combine(Tables)
in Out

5. Isolating failures (one bad object does not kill refresh)

Wrap each call:

Result = try SF_RunSoql(soql) otherwise #table({"Error"}, {{"SOQL failed"}})

Then filter or flag error rows in a diagnostics table for the report.

6. Performance and crashes

  • Do not fire all objects in parallel inside one query unless you control concurrency — prefer sequential List.Transform or a small custom loop with a low page cap.
  • Select only columns you need; wide SOQL plus Table.FromRecords is a common memory spike.
  • Keep token in one query; reference it read-only from every pull.
  • Use the Advanced · DAX · Settings tab in queries-job-aide.html for Desktop data-load options.
