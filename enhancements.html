Task 2 — POC Master Records: Child Match Enhancements

Direct URL Salesforce Connection · AccountIds from POC Master spreadsheet · No new objects/fields

🔗
Critical

Bulk REST API Chunking (200-ID Batches)

Salesforce REST API returns max 2,000 records per call and SOQL IN clauses cap at ~10,000 characters. Chunk the POC Master AccountIds into batches of 200, fire parallel API calls, then union results in Power BI. Prevents query timeout and governor limit hits.

  • Split AccountId list into 200-row chunks using List.Split
  • Use List.Transform to build per-chunk SOQL strings
  • Combine with Table.Combine post-fetch
  • Add retry logic on HTTP 429 / 503 responses
Impact: Prevents total query failure on large ID sets
🔍
Critical

Fuzzy AccountId Normalization

POC Master IDs may contain 15-char vs 18-char Salesforce IDs, mixed case, or leading/trailing whitespace. Normalize all IDs to uppercase 18-char before matching to avoid missed joins that silently drop valid records.

  • Trim whitespace: Text.Trim
  • Pad 15→18 char: custom M function using checksum algorithm
  • Uppercase: Text.Upper
  • Flag anomalies: IDs not matching 15/18-char length
Impact: Eliminates silent data loss from ID format mismatches
📊
Critical

Match Rate Scorecard per Child Object

For each child object (Contact, Opportunity, Case, Contract, etc.) compute: Total AccountIds in POC Master vs. count with ≥1 child record. Surface as a match-rate % KPI card per object. Immediately shows gaps for management review.

  • Matched / Unmatched / Partial groupings
  • Zero-child-record accounts flagged as orphans
  • Trend line if historical snapshots exist
  • Drill-through to the specific unmatched account list
Impact: Executive-ready gap analysis in one view
🌲
High

Account Hierarchy Traversal

If a POC Master AccountId is a parent account, child records may be attached to child accounts not in the spreadsheet. Add a recursive parent-child lookup using Account.ParentId to capture the full account tree without creating new fields.

  • One extra REST call per parent to get child account IDs
  • Max 5 levels deep (Salesforce hierarchy limit)
  • Flag records found via hierarchy vs. direct match
Impact: Finds hidden child records missed by flat ID match
⏱️
High

Delta Change Detection

Cache the last-run snapshot in a Power BI dataflow or local CSV. On each refresh, compare new child record counts vs. prior run. Surface net-new, removed, and modified child records per object type. Critical for audit trails.

  • Use LastModifiedDate > [last_refresh] in SOQL WHERE clause
  • Store snapshot date as a parameter in Power BI
  • Delta badges on KPI cards (▲ +12 / ▼ -3)
Impact: Converts static snapshot into a living audit tool
🗂️
High

Multi-Object Relationship Matrix

Build a cross-object presence matrix: rows = AccountIds, columns = child object types. Cell value = record count. Enables instant visual identification of accounts that have Contacts but no Opportunities, or Cases with no Contracts.

  • Pivot child-object counts per account
  • Conditional formatting: red=0, amber=1-2, green=3+
  • Sortable by any object column
  • Exportable to Excel for offline review
Impact: Single-page cross-object coverage view
🚨
High

Orphan Record Detection & Alerting

Identify AccountIds in the POC Master spreadsheet that have zero child records across ALL objects — true orphans. Surface as a dedicated page with account details. These likely represent data quality issues or accounts pending migration.

  • COUNT(ALL child objects) = 0 filter
  • Show account name, type, created date, owner
  • One-click export of orphan list
  • Power BI alert if orphan count exceeds threshold
Impact: Directly actionable data quality output for management
🔐
Medium

OAuth 2.0 Connected App Token Refresh

Direct URL connection uses OAuth 2.0. Implement automatic token refresh in M query using Web.Contents with a token endpoint call at the top of each query. Avoids manual re-auth breaking scheduled refreshes.

  • Store Client ID / Secret in Power BI gateway credentials
  • Token request via /services/oauth2/token
  • Pass Bearer token in all subsequent SOQL REST calls
  • Use Power BI dataflow for centralized credential management
Impact: Unbreakable scheduled refresh without manual intervention
📅
Medium

Activity Timeline Cross-Object View

Merge Task, Event, EmailMessage, and CaseComment records for each AccountId into a unified chronological timeline. Provides a 360° activity view for any selected account without creating new fields.

  • Use existing WhatId / WhoId / AccountId relationships
  • Union Tasks + Events + Cases by date
  • Color-code by object type in visual
  • Filter by date range and activity type
Impact: Deep account intelligence view for relationship managers
📈
Medium

Data Quality Scoring per Account

Score each POC Master account 0–100 based on completeness across child objects: Contact present (+20), Opportunity present (+20), Contract signed (+20), Cases resolved (+20), Activities logged (+20). Surfaces data health at a glance.

  • Calculated column in Power BI data model
  • Gauge visual per account in drill-through
  • Distribution histogram across all accounts
  • Sort/filter by score band
Impact: Quantified data quality metric for management reporting
⚡
Medium

Incremental Refresh on Child Object Queries

Configure Power BI incremental refresh policy using RangeStart and RangeEnd parameters in each child-object SOQL query's WHERE clause. Reduces refresh time from full-reload to delta-only on large objects like Task and Event.

  • Add LastModifiedDate >= @RangeStart AND LastModifiedDate < @RangeEnd
  • Set 1-day incremental, 30-day historical window
  • Only applicable to Premium capacity workspaces
Impact: 80%+ faster refresh on high-volume child objects
🖨️
Low

One-Click PDF Export per Account

Add a bookmark-based "Account Profile" page that consolidates all child record counts, activity timeline, and data quality score for one account. Use Power BI's built-in Export to PDF. Print-ready layout for relationship managers.

  • Single-account filter via slicer or drill-through
  • TD branding header / footer on the page
  • Date-stamped export for audit evidence
Impact: Client-ready output without leaving Power BI

  • Filter by LOB and date range
Impact: Audit-ready permission change log — critical for compliance review

Task 2 — Salesforce Direct URL Connection Setup

Power BI → Salesforce REST API via OAuth 2.0 Connected App · No new objects or fields

🔧 Step-by-Step: Connected App Setup

1

In Salesforce Setup → App Manager → New Connected App. Enable OAuth, add scopes: api, refresh_token, offline_access

2

Copy Consumer Key (Client ID) and Consumer Secret. Store in Power BI gateway credentials — never in query text.

3

In Power BI, use Web connector with Base URL: https://[instance].salesforce.com. Token endpoint: /services/oauth2/token

4

Schedule gateway refresh — OAuth token auto-renews. Test with a simple SOQL: SELECT Id, Name FROM Account LIMIT 1

📝 Base M Query Template — SOQL via REST API

// Task 2: Base REST API Query Function let InstanceUrl = "https://yourorg.salesforce.com", TokenUrl = InstanceUrl & "/services/oauth2/token", // Get OAuth Token TokenBody = "grant_type=password" & "&client_id=" & ClientId & "&client_secret=" & ClientSecret & "&username=" & SFUser & "&password=" & SFPass, TokenResponse = Json.Document( Web.Contents(TokenUrl, [ Content = Text.ToBinary(TokenBody), Headers = [#"Content-Type"="application/x-www-form-urlencoded"] ]) ), AccessToken = TokenResponse[access_token], // Execute SOQL SOQL = "SELECT Id, Name FROM Account LIMIT 200", EncodedSOQL = Uri.EscapeDataString(SOQL), QueryUrl = InstanceUrl & "/services/data/v59.0/query/?q=" & EncodedSOQL, Result = Json.Document( Web.Contents(QueryUrl, [ Headers = [Authorization = "Bearer " & AccessToken] ]) ), Records = Result[records] in Records
